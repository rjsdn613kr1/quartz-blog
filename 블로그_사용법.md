## 글 쓰는 법

### 1. Obsidian에서 새 파일 생성
Obsidian을 열면 볼트가 `quartz-blog/content/` 로 설정되어 있습니다.
여기서 새 파일을 만들면 됩니다.

### 2. 파일 상단에 frontmatter 작성
```
---
title: 글 제목
date: 2026-04-02
tags:
  - 태그1
  - 태그2
draft: false
description: 글 요약 (목록에 표시됨)
---

여기서부터 본문
```

| 항목 | 설명 | 필수 여부 |
|------|------|----------|
| `title` | 글 제목 | 권장 |
| `date` | 작성일 | 권장 |
| `tags` | 태그 | 선택 |
| `draft: true` | 비공개 처리 (블로그에 안 올라감) | 선택 |
| `description` | 글 요약 | 선택 |

### 3. 저장하면 끝
저장 후 5분 이내에 Obsidian Git이 자동으로 GitHub에 올립니다.
GitHub에 올라가면 약 2분 후 블로그에 자동 반영됩니다.

**총 소요시간: 최대 7분**

## 카테고리 분류 (폴더 사용)

`content/` 안에 폴더를 만들면 자동으로 카테고리가 됩니다.

```
content/
├── index.md
├── 에세이/
│   ├── 첫번째글.md
│   └── 두번째글.md
├── 스터디/
│   └── 파이썬공부.md
└── 회고/
    └── 2026년회고.md
```

폴더 안에 `index.md`를 만들면 카테고리 소개 페이지도 만들 수 있습니다.

## 배포 흐름 (자동)

```
Obsidian에서 글 작성/수정
        ↓ (5분마다 자동)
Obsidian Git 플러그인이 GitHub push
        ↓ (즉시 자동)
GitHub Actions 빌드 시작 (~2분)
        ↓
블로그 자동 업데이트 완료
```

### 배포 상태 확인하는 법
https://github.com/rjsdn613kr1/quartz-blog/actions
여기서 초록색 체크 = 배포 완료, 노란색 = 진행 중, 빨간색 = 오류

### 즉시 반영하고 싶을 때
Obsidian에서 `Cmd+P` → `Obsidian Git: Create backup` 실행
(5분 기다리지 않고 바로 push)

## 블로그 커스터마이징

### 제목 변경
`quartz.config.ts` 파일에서:
```
pageTitle: "원하는 제목"
```

### 홈페이지 내용 변경
`content/index.md` 파일을 수정하면 됩니다.

### 색상 변경
`quartz.config.ts`의 `colors` 섹션에서 수정:
- `light`: 배경색
- `dark`: 글자색
- `secondary`: 링크/강조색
- `lightMode` = 라이트 모드, `darkMode` = 다크 모드

### 수정 후 반영
`quartz.config.ts`, `quartz.layout.ts` 등 설정 파일은 Obsidian이 아닌
VSCode 등 텍스트 에디터로 수정해야 합니다.
수정 후 저장하면 Obsidian Git이 자동으로 감지해서 push → 자동 배포됩니다.

## 비공개 글 작성법

frontmatter에 `draft: true` 추가:
```
---
title: 아직 공개 안 할 글
draft: true
---
```
→ GitHub에는 올라가지만 블로그에는 표시 안 됨

또는 `content/private/` 폴더에 넣으면 통째로 비공개 처리됩니다.

## 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 글이 블로그에 안 보임 | 아직 배포 안 됨 | 7분 기다리기 |
| 글이 계속 안 보임 | `draft: true` 설정됨 | frontmatter 확인 |
| 배포 실패 | GitHub Actions 오류 | actions 페이지에서 로그 확인 |
| Obsidian Git 안 됨 | 인터넷 연결 또는 설정 문제 | `Cmd+P` → Create backup 수동 실행 |
